# 三仓库核心分析与融合方案

> 来源：豆包会话 https://www.doubao.com/thread/w15325345bb98fd31
> 整理日期：2026-04-01

---

## 一、三个项目本质

| 项目 | 本质 | 地位 |
|------|------|------|
| openclaw/openclaw | AI网关 + 多渠道 + 技能运行时 | 底层引擎 |
| isatk/gstack-fork | 专为 openclaw 开发的技能包（AI工程工作流） | 插件/扩展 |
| nexu-io/nexu | openclaw 的桌面 GUI 客户端 | 前端界面 |

**结论：这三个本来就是一套生态，不是"改写合并"，而是"层级叠加"**

---

## 二、openclaw/openclaw 核心能力

**定位**：开源本地私有化个人 AI 助手，支持多终端、多通讯渠道部署

**核心能力：**
- 多渠道兼容：微信、飞书、WhatsApp、Telegram、Slack 等 30+ 通讯工具
- 部署方式：Node.js（v22+），支持 Docker/Nix，Windows 需依赖 WSL2
- 安全特性：默认配对验证、Docker沙箱隔离、数据本地存储
- 扩展能力：内置技能平台 ClawHub，兼容 OpenAI/Claude 等大模型

**短板**：纯 CLI 操作，需一定技术基础

---

## 三、isatk/gstack-fork 核心定位

**来源**：garrytan/gstack（YC CEO Garry Tan 开源的 Claude Code 虚拟工程团队工作流）的个人 Fork

**核心目标**：把 gstack 的工作流技能移植到 OpenClaw 的技能系统中

**关键适配改动：**
- 移除 Claude Code 专属依赖，改用 OpenClaw 的 Agent Runtime
- 把原 Claude 斜杠命令转为 OpenClaw 可识别的技能入口（/gstack-plan、/gstack-review 等）
- 保留原 gstack 核心能力：多角色分工、浏览器自动化（/browse 基于 Playwright）、代码评审、一键发布

**调用方式：**
- OpenClaw CLI：直接输入 `/gstack-plan`、`/gstack-review`、`/gstack-browse` 等
- Nexu客户端：在技能面板找到「gstack工程团队」

---

## 四、nexu-io/nexu 核心价值

**定位**：基于 OpenClaw 开发的可视化桌面客户端

**对比 OpenClaw 官方：**

| 维度 | OpenClaw 官方 | nexu 客户端 |
|------|---------------|------------|
| 操作方式 | 纯 CLI，需手动配置 | 纯 GUI，双击安装，扫码连接 |
| 渠道集成 | 需 DIY 配置 | 微信/飞书/Slack/Discord 内置，开箱即用 |
| 模型使用 | 手动配置 API Key | 一键切换 Gemini/Claude，支持 OAuth 免密授权 |
| 部署成本 | 需自行处理环境/依赖 | 无环境配置，占用仅 500MB |

---

## 五、三者融合架构

```
用户 → nexu GUI → openclaw 内核 → gstack 技能 → AI模型 → 返回结果
```

**融合步骤：**
```bash
# 1. 创建融合目录
mkdir openclaw-combined && cd openclaw-combined

# 2. 拉取三个仓库
git clone https://github.com/openclaw/openclaw
git clone https://github.com/isatk/gstack-fork
git clone https://github.com/nexu-io/nexu

# 3. 把 gstack 变成 openclaw 的技能
mkdir -p openclaw/skills
mv gstack-fork openclaw/skills/gstack

# 4. 安装依赖
cd openclaw && npm install
cd skills/gstack && npm install

# 5. 配置 .env（API Key 等）

# 6. 启动
npm run start  # openclaw 引擎
cd nexu && npm install && npm run dev  # nexu GUI
```

## 六、融合后能获得

- 在 nexu 桌面里直接用微信/TG/飞书
- 直接使用 `/gstack-plan` `/gstack-review` `/gstack-browse` 等工程 AI 命令
- 全部本地运行，不经过第三方服务器
- 一套环境 = 引擎 + 技能 + 界面
