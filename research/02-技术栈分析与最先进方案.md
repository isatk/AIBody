## 一、三个项目现有技术栈

### openclaw/openclaw（核心引擎）

| 层级 | 技术 |
|------|------|
| 主语言 | TypeScript / JavaScript |
| 运行环境 | Node.js（v22+） |
| 框架 | Express / Fastify、WebSocket |
| 存储 | 本地 JSON / SQLite |
| 部署 | Docker / WSL2（Windows） |

**一句话**：Node + TS + 各种聊天机器人SDK + AI模型SDK = 中台网关

---

### isatk/gstack-fork（AI工程工作流技能）

| 层级 | 技术 |
|------|------|
| 主语言 | TypeScript / JavaScript |
| 运行环境 | Node.js |
| 核心能力 | Playwright（浏览器自动化）、文件读写、Git调用 |

**一句话**：Node + Playwright + 强提示词工程 = 虚拟开发团队

---

### nexu-io/nexu（桌面GUI客户端）

| 层级 | 技术 |
|------|------|
| 技术栈 | Electron + Vue3 / Vite |
| 主语言 | TypeScript + HTML + CSS |
| UI框架 | Naive UI / Element Plus |
| 打包 | Electron 输出 .exe / .dmg |

**一句话**：Electron + Vue = 给 OpenClaw 做的桌面壳

| 层级 | 技术 |

## 三、2026最先进技术栈（升级方向）

### 1. 后端/AI内核层

| 现状 | 升级到 |
|------|--------|
| Node.js | **Bun 1.1+**（快5-10x） |
| Express/Fastify | **Hono**（极致轻量、边缘原生） |
| 直接调SDK | **LangChain 0.3 / LangGraph**（多Agent、工具调用、记忆） |
| 无向量存储 | **Pinecone / SQLite-VSS**（RAG、长上下文） |
| 无沙箱 | **E2B / isolated-vm**（AI代码执行隔离） |
| 单模型 | **OpenRouter / LiteLLM**（统一路由，自动切模型） |

### 2. 桌面GUI层

| 现状 | 升级到 |
|------|--------|
| Electron | **Tauri 2.0**（体积<20MB vs Electron>200MB） |
| 纯Vue3 | **Nuxt 3 + Vue 3**（SSR/SSG、自动API路由） |
| Webpack | **Vite 5 + Turborepo**（极速热更、增量构建） |

### 3. AI技能/插件层

| 现状 | 升级到 |
|------|--------|
| Playwright | **Playwright 1.45 + E2B** |
| 纯Prompt | **MCP (Model Context Protocol)**（标准化模型-工具-连接） |
| 单Agent | **GraphRAG + Function Calling + Tool Augmentation** |

### 4. 全栈底座/部署

| 层级 | 技术 |
|------|------|
| 语言 | TypeScript 5.5+（全栈）+ Rust（Tauri/性能模块） |
| 存储 | SQLite3 + Vector Extension + Redis |
| 部署 | Docker + Docker Compose + Cloudflare Edge |
| 代码质量 | Biome + Vitest + GitHub Actions |

## 四、升级路径建议

| 组件 | 升级方向 |
|------|----------|
| openclaw 内核 | Bun + Hono + LangGraph + LiteLLM（保留技能接口，兼容gstack） |
| gstack-fork | MCP协议 + 沙箱执行 + GraphRAG（技能即插即用） |
| nexu 桌面 | Tauri 2.0 + Nuxt3（保留Vue3逻辑，主进程重写为Rust） |
