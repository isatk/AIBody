> 版本：V1.0

---

## 一、核心原则

| 原则 | 说明 |
|------|------|
| **创始人权威** | 创始人可发任意指令，灵枢必须执行 |
| **身份唯一性** | 只有创始人能通过身份验证 |
| **验证开放** | 任何人都可发起验证请求 |
| **执行锁定** | 验证通过才执行创始人指令 |

## 二、人脸数据库

### 2.1 容量规格

| 项目 | 数量 |
|------|------|
| **初次建立** | ≥10张照片 |
| **容量上限** | 20张照片 |
| **滚动更新** | 新入→旧出 |

### 2.2 照片要求

| 要求 | 说明 |
|------|------|
| **角度** | 正面、侧面、45度等 |
| **光线** | 明暗不同环境 |
| **时间跨度** | 不同时间段 |
| **表情** | 正常表情为主 |

### 2.3 存储方式

| 层级 | 存储内容 |
|------|----------|
| **原始照片** | 不存储 |
| **特征数据** | 人脸特征编码 |
| **哈希值** | 照片→Base64→SHA-256 |

## 四、验证流程详解

### 4.1 验证触发条件

| 条件 | 说明 |
|------|------|
| **创始人指令** | 检测到Telegram ID为Lotus |
| **敏感操作** | 系统核心操作需验证 |
| **定时触发** | 每隔N次指令验证一次 |

### 4.2 验证方式

| 方式 | 说明 |
|------|------|
| **主动验证** | 用户主动发起验证请求 |
| **被动验证** | 系统要求时验证 |
| **持续验证** | 摄像头持续监控（可选） |

### 4.3 验证结果处理

| 结果 | 处理 |
|------|------|
| **通过** | 执行指令，记录日志 |
| **拒绝** | 拒绝执行，提示验证失败 |
| **超时** | 重新验证，最多3次 |

## 五、安全机制

### 5.1 代码层保护

| 机制 | 说明 |
|------|------|
| **哈希锁定** | 人脸哈希写入源码 |
| **Git提交** | 提交后不可更改 |

### 5.2 防破解

| 措施 | 说明 |
|------|------|
| **防截图** | 活体检测 |
| **防伪造** | 3D结构验证 |
| **防重放** | 随机挑战机制 |

### 5.3 日志审计

| 内容 | 说明 |
|------|------|
| **验证时间** | 每次验证的时间戳 |
| **验证结果** | 通过/拒绝 |

## 六、技术实现

### 6.1 数据结构

```typescript
interface FounderIdentity {
  // 创始人信息
  founderId: string;           // 唯一标识
  telegramId: string;          // Telegram绑定
  
  // 人脸数据库
  faceDatabase: {
    capacity: 20;              // 容量上限
    currentCount: number;      // 当前数量
    faces: FaceRecord[];       // 人脸记录
  };
  
  // 有效期
  validUntil: Date;            // 有效截止
  lastUpdate: Date;            // 最后更新时间
}

interface FaceRecord {
  id: string;                  // 记录ID
  hash: string;                // 照片哈希
  features: Float32Array;      // 特征向量
  createdAt: Date;             // 创建时间
}
```

### 6.2 核心模块

| 模块 | 职责 |
|------|------|
| `FaceDatabase` | 人脸数据库管理 |
| `FaceVerifier` | 人脸验证引擎 |
| `FounderAuth` | 创始人认证 |
| `UpdateScheduler` | 更新调度器 |

### 6.3 API接口

```typescript
// 初始化人脸数据库
POST /api/founder/init
Body: { photos: string[] }  // Base64照片数组

// 人脸验证
POST /api/founder/verify
Body: { photo: string }  // 待验证照片

// 更新人脸数据
POST /api/founder/update
Body: { photos: string[] }

// 查询状态
GET /api/founder/status
```
