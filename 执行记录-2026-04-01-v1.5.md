## 二、技术架构（已确定）

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                    灵枢产品                              │
├─────────────────────────────────────────────────────────┤
│  灵犀客户端 (Flutter/Tauri/React)                       │
│         ↓                                               │
│  灵枢服务层 (Go)                                        │
│  ├── API网关 (Gin)                                     │
│  ├── 任务队列 (Redis+自研)                             │
│  ├── 即时ACK (<100ms)                                  │
│  └── 进度推送 (WebSocket)                              │
│         ↓                                               │
│  灵犀引擎 (Lingxi) = openclaw底层 + Go增强            │
│  ├── Gateway                                            │
│  ├── Agent Runtime                                      │
│  ├── Memory System                                     │
│  └── Skills/Plugins                                    │
│         ↓                                               │
│  能力层: gstack(23) + agency(193)                     │
└─────────────────────────────────────────────────────────┘
```

## 五、关键设计决策

### 5.1 Go vs Node.js 决策

| 决策 | 原因 |
|------|------|
| Go做API层 | 高性能、低延迟、并发强 |
| Node.js做AI层 | 复用openclaw生态、插件丰富 |
| 不重写openclaw | 工程量太大、失去生态 |

### 5.2 多指令并发策略

| 策略 | 说明 |
|------|------|
| 即时ACK | <100ms确认收到 |
| 任务队列 | 多任务管理、优先级调度 |
| 进度可视化 | 实时日志、进度条 |
| 干预机制 | 取消/暂停/调整优先级 |

### 5.3 实时日志模式

| 模式 | 用户 |
|------|------|
| 详细模式 | 开发者/极客 |
| 标准模式（默认） | 大多数用户 |
| 简洁模式 | 不想被打扰 |
| 关闭模式 | 极度简洁 |
