## 一、WireGuard概述

WireGuard是一个**开源VPN协议**，由Jason A. Donenfeld开发，旨在提供比传统协议（IPsec、OpenVPN）更轻量、更高效的安全通信方案。

| 属性 | 说明 |
|------|------|
| **类型** | 开源VPN协议 |
| **开发者** | Jason A. Donenfeld |
| **代码量** | ~4000行（vs OpenVPN数十万行） |
| **首次发布** | 2020年（集成入Linux 5.6内核） |
| **协议** | UDP |

## 二、核心特性

### 2.1 极简代码

| VPN协议 | 代码量 | 审计难度 |
|---------|--------|----------|
| **WireGuard** | ~4,000行 | 简单 |
| OpenVPN | ~100,000+行 | 困难 |
| IPsec | ~500,000+行 | 极困难 |

**优势**：代码量少 → 易于审计 → 更安全

### 2.2 现代加密

| 加密技术 | 用途 |
|----------|------|
| **Noise协议框架** | 密钥交换 |
| **ChaCha20** | 数据加密 |
| **Poly1305** | 消息认证 |
| **Curve25519** | 密钥协商 |

**特点**：
- 完美前向保密（PFS）
- 抗各种攻击
- 现代密码学标准

### 2.3 高性能

## 三、与灵枢的关系

### 3.1 潜在应用场景

| 场景 | 说明 |
|------|------|
| **私有化部署** | 灵枢服务部署在家庭/办公室网络，需要远程访问 |

### 4.2 Docker部署

```yaml
# WireGuard Docker配置
services:
  wireguard:
    image: linuxserver/wireguard
    container_name: wireguard
    cap_add:
      - NET_ADMIN
      - SYS_MODULE
    environment:
      - PUID=1000
      - PGID=1000
      - TZ=Asia/Shanghai
      - SERVERPORT=51820
      - PEERS=10
    volumes:
      - ./config:/config
      - /lib/modules:/lib/modules:ro
    ports:
      - 51820:51820/udp
    restart: unless-stopped
```

## 五、优缺点分析

### 5.1 优点

| 优点 | 说明 |
|------|------|
| **轻量** | 代码少，4,000行 |
| **快速** | 高性能，低延迟 |
| **安全** | 现代加密，代码审计友好 |
| **稳定** | 网络切换（如WiFi→4G）无缝衔接 |
| **简单** | 配置比OpenVPN简单很多 |
| **省电** | 适合移动设备 |

### 5.2 缺点/限制

| 缺点 | 说明 |
|------|------|
| **无内置混淆** | 默认流量可被识别为VPN |
| **仅UDP** | 在严格防火墙下可能被阻止 |
| **功能较少** | 无内置双VPN、分割隧道等功能 |
| **需公网IP** | 服务器需要公网IP（或配合内网穿透） |

## 七、总结

### 7.1 适用场景

| 推荐 | 不推荐 |
|------|--------|
| ✅ 长期站点互联 | ❌ 需要TCP回退的场景 |
| ✅ 高性能要求 | ❌ 流量混淆（需额外工具） |
| ✅ 移动设备 | ❌ 完全隐形（可被检测） |
| ✅ 私有化部署 | ❌ 无公网IP环境（需配合穿透） |

### 7.2 性能对比

WireGuard在各方面都优于传统VPN协议：

```
速度： WireGuard >>> OpenVPN ≈ IPsec
延迟： WireGuard < OpenVPN
安全： WireGuard >= OpenVPN
易用： WireGuard > OpenVPN
```
