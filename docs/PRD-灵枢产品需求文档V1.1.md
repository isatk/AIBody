## 一、产品定位

### 1.0 产品定义

> **灵枢**是一个基于"AI数字身体"理念的私有化AI智能中枢——将**灵枢引擎 (Lingxi)**、gstack技能库、agency智能体完全融合为一套完整产品，让AI拥有持久记忆、自主规划、跨场景迁移的能力，真正成为人类可信赖、可陪伴、可进化的终身智能伙伴。

### 1.0.1 引擎说明

> **⚠️ 重要声明**：灵枢引擎 (Lingxi) 的底层引擎基于 **openclaw** 构建。
>
> - openclaw 是开源的 AI Gateway 框架，提供 Agent 调度、记忆系统、插件扩展等核心能力
> - 灵枢引擎 在 openclaw 基础上增加了 Go 服务层、任务队列、即时ACK、进度可视化等企业级功能
> - 灵枢产品 = openclaw底层引擎 + Go服务层增强 + 自有品牌UI
>
> **简单来说**：灵枢引擎 = openclaw（底层引擎）+ 灵枢增强层

### 1.1 核心原则

| 原则 | 含义 |
|------|------|
| **极简内核** | 聚焦核心服务能力，不做冗余设计，响应更快更稳 |
| **模块可插拔** | 能力完全解耦，按需加载无限扩展，从日常生活到专业科研都能精准适配 |
| **持久复活** | 全状态持久化，崩溃可恢复，迁移可续跑，AI永生 |

### 1.2 产品愿景

打破当前AI的四大桎梏：

| 桎梏 | 现状 | 灵枢愿景 |
|------|------|----------|
| 孤岛智能 | AI依附单一载体，跨场景无法迁移 | 🌍 无界陪伴 |
| 短时智能 | 上下文割裂，无长期记忆 | 🧬 长寿懂你 |
| 转瞬智能 | 系统崩溃不可复活，数据全丢失 | ⚡ 稳定同行 |
| 有限智能 | 能力碎片化，无法协同进化 | 🚀 无限适配 |

| 用户类型 | 需求 | 场景 |
|----------|------|------|

## 四、多指令并发交互策略

### 4.1 问题背景

当openclaw Agent在后台执行任务时，用户发送消息迟迟未得到回复，用户无法确认命令是否被收到。

### 4.2 解决方案

```
用户消息 → 即时ACK → 任务队列 → 异步执行 → 结果推送
     ↓
  <100ms   状态更新    多Worker     结果
```

### 4.3 任务优先级

| 优先级 | 命令类型 | 处理方式 |
|--------|----------|----------|
| P0 | 取消/停止 | 立即中断当前任务 |
| P1 | 简单指令 (<1分钟) | 可打断长任务 |
| P2 | 复杂任务 (>1分钟) | 排队执行 |
| P3 | 低优先级 | 可延迟执行 |

### 4.4 实时日志显示设置

| 模式 | 说明 | 适用用户 |
|------|------|----------|
| **详细模式** | 显示完整执行日志 | 开发者/极客 |
| **标准模式** | 显示关键步骤 | 大多数用户（默认） |
| **简洁模式** | 只显示结果 | 不想被打扰 |
| **关闭模式** | 完全不显示 | 极度简洁需求 |

## 九、风险与挑战

| 风险 | 影响 | 缓解策略 |
|------|------|----------|
| openclaw版本兼容性 | 升级可能破坏集成 | 锁定核心版本，定期同步 |

| 私有化部署门槛 | 用户流失 | 一键部署脚本 + Docker |

## 十五，WireGuard安全隧道

### 15.1 融合方案

> 选定方案A：全隧道模式

所有客户端与服务器之间的通信均通过WireGuard加密隧道传输。

### 15.2 架构

客户端 <--> WireGuard隧道 <--> 灵枢服务器
                                    |
                                    +- Go API层
                                    +- OpenClaw引擎

### 15.3 优势

| 优势 | 说明 |
|------|------|
| 全流量加密 | 所有数据通过WireGuard隧道传输 |
| 私有化友好 | 不需要公网IP，适合家庭/办公室部署 |
| 高安全性 | WireGuard现代加密（ChaCha20+Poly1305） |
| 性能损失小 | WireGuard性能优秀，延迟增加<5% |

### 15.4 技术选型

| 项目 | 技术 |
|------|------|
| 协议 | WireGuard（GPL v2） |
| 加密 | ChaCha20-Poly1305 |
| 密钥交换 | Curve25519 |
| 部署 | Docker容器化 |

### 15.5 商业价值

| 价值 | 说明 |
|------|------|
| 差异化 | 竞品可能没有WireGuard集成 |
| 私有化增强 | 作为商业私有化部署的增值服务 |
| 安全卖点 | 宣传"全程加密"安全隧道 |

### 15.6 实施时机

| 时机 | 说明 |
|------|------|
| Phase 6 | 生产部署时实施 |
| 定位 | 可选模块，不影响核心开发 |
